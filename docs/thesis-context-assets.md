## 2. 项目技术栈（基于 package.json 与实际用法）

### 2.1 前端基础框架

- **Next.js 16（App Router）**：承载路由、布局与页面组织。
- **React 19 + TypeScript 5**：组件化开发与类型安全。
- **Tailwind CSS 4**：原子化样式与打印态样式协同（`@media print`）。

### 2.2 UI 与交互能力栈

- **HeroUI v3**（`@heroui/react`、`@heroui/styles`）：页面与表单主 UI 组件体系。
- **lucide-react**：图标系统。
- **@dnd-kit**：模块与信息层拖拽排序。
- **@internationalized/date + HeroUI DatePicker**：日期输入与日历交互。

### 2.3 数据与状态管理

- **@tanstack/react-query**：服务端状态拉取、缓存与失效控制。
- **Zustand**：客户端状态管理（用户态等本地状态）。
- **本地存储封装（`src/lib/storage`）**：编辑偏好持久化（面板宽度、AI 模式等）。

### 2.4 编辑器与 AI 相关能力

- **Tiptap 3**（`@tiptap/*`）：富文本信息层编辑能力。
- **SSE 流式封装**（`src/lib/request.ts`、`src/lib/ai/stream-chat.ts`）：AI 流式对话。
- **Markdown 渲染**（`react-markdown` + `remark-gfm` + `rehype-sanitize`）：AI 回复内容安全展示。

### 2.5 导出与截图能力

- **浏览器打印导出（`window.print` + `@media print`）**：简历 PDF 导出。
- **html2canvas-pro**：简历列表封面截图生成。

## 3. 项目定位与问题背景（可直接改写进绪论）

### 2.1 项目定位

- 项目类型：面向求职场景的在线简历编辑系统（前端工程）。
- 核心目标：降低简历结构化编辑门槛，提高内容生成与修改效率，并支持可导出的版式化结果（PDF）。
- 交互形态：三栏编辑器（AI 对话 / 中央画布 / 模块管理与诊断）。

### 2.2 待解决问题

- 传统简历编辑流程中，内容结构与展示样式耦合，导致复用性与可维护性弱。
- 用户在“模板管理-内容录入-版式预览-导出”链路中频繁切换，操作成本高。
- AI 能力接入后，多轮会话、流式响应、输入法兼容、上下文持久化等工程问题会显著增加复杂度。

### 2.3 解决思路（系统主线）

- 以“Section（模块）+ Info Layer（信息层）”构建可组合的简历结构模型。
- 在 Builder 页面中实现“结构管理 + 内容编辑 + 即时预览 + AI 辅助”闭环。
- 通过 API 类型化封装、查询缓存与本地偏好持久化，保证可维护性与交互连续性。

证据来源：`2026-04-21.md`、`2026-04-23.md`、`2026-04-24.md`、`2026-04-26.md`、`2026-04-30.md`、`2026-05-04.md`。

## 4. 系统架构与代码组织（基于 src 目录）

### 4.1 架构风格

- 整体采用“**前端分层 + 领域建模**”架构：页面层、组件层、API 层、业务模型层、工具层分离。
- 页面以 Next.js App Router 为骨架，Builder 场景通过路由分段与组件拆分实现三栏协同。
- 类型体系分为 API 契约类型与业务实体类型，降低接口变更对 UI 的直接冲击。

### 4.2 `src/` 目录分层职责

- **`src/app`**：路由页面与场景编排（`dashboard`、`builder`、`auth` 等）。
- **`src/components`**：跨页面通用组件与信息层渲染组件（如 `section-render`）。
- **`src/apis`**：HTTP API 薄封装，统一请求入口与返回类型。
- **`src/types/api`**：与后端接口一一对应的请求/响应类型。
- **`src/types/business`**：前端业务实体（Resume/Section/Content 等领域对象）。
- **`src/lib`**：通用基础设施（request、SSE、storage、builder key、截图与导出辅助）。
- **`src/stores`**：基于 Zustand 的客户端状态容器。
- **`src/styles`**：全局样式与主题扩展。

### 4.3 运行时数据流（论文可画图）

- 页面层发起交互（Builder/ Dashboard）。
- API 层通过 `src/apis/*` 调用后端接口，`src/lib/request.ts` 统一请求行为。
- React Query 缓存接口结果并驱动页面更新，关键操作通过 `invalidateQueries` 或乐观更新同步 UI。
- 组件层消费 `types/business` 领域模型渲染视图，必要时将编辑结果转换为 `types/api` 请求体提交。

## 6. 核心设计资产（可用于“系统设计与实现”章节）

### 4.1 领域模型资产

- **Resume（简历）**：承载标题、个人信息、模块列表、JD 等业务实体。
- **Section（模块）**：按 `order` 排序，关联一个内容模板并承载具体内容。
- **Info Layer（信息层）**：模板最小展示单元（三种固定类型），支持字段重命名与内容录入。
- **Content/Info**：模块内可重复内容项及其多字段值集合。

证据来源：`2026-04-22.md`、`2026-04-23.md`、`2026-04-24.md`。

### 4.2 页面与交互架构资产

- **Dashboard/Resumes**：简历列表、创建入口、卡片、分页、删除。
- **Builder 顶栏**：返回、标题状态、保存/导出、面板开关。
- **Builder 三栏**：
  - 左栏：AI 对话、会话历史、模式切换、联网搜索、JD Modal。
  - 中栏：A4 画布预览、个人信息模块、自定义模块编辑。
  - 右栏：模块管理、添加/排序/删除、设计/诊断入口。

证据来源：`2026-04-23.md`、`2026-04-24.md`、`2026-04-26.md`、`2026-04-30.md`、`2026-05-04.md`。

### 4.3 编辑器与展示一致性资产

- 富文本信息层支持编辑/只读切换、工具栏能力、段落缩进扩展与空段清理。
- 统一信息层“展示态/编辑态”行为，空值提示、字段对齐与交互样式一致。
- 自定义模块在中心画布中支持点击进入编辑、模块外点击保存退出。

## 7. 技术决策与工程问题闭环（可用于“关键技术与难点”）

### 5.1 模块排序持久化策略

- 问题：本地拖拽排序如果不落库，刷新后顺序回滚。
- 方案：接入 `/section/reorder`，采用“乐观更新 -> 成功以服务端顺序覆盖 -> 失败回滚”。
- 价值：提升排序交互即时性与最终一致性。

证据来源：`2026-04-23.md`（23:19）。

### 5.2 数据源拆分与 Query Key 规范化

- 问题：`resume/detail` 不再返回 `sections` 后，页面状态依赖链断裂风险增大。
- 方案：新增 `resumeQueryKey` / `resumeSectionsQueryKey`，将 `resume/detail` 与 `section/list` 解耦并分别缓存。
- 价值：降低接口变更对页面结构的冲击，提升可维护性。

证据来源：`2026-04-23.md`（00:32）。

### 5.3 AI 流式对话接入

- 问题：SSE 解析、首条消息建会话、历史回放与流式并发覆盖冲突。
- 方案：封装 `postSse` + `streamAiChat`；引入会话草稿态、跳过下一帧历史加载、按 `purpose` 对齐请求。
- 价值：实现稳定的多轮会话体验与流式反馈。

证据来源：`2026-04-26.md`、`2026-04-27.md`。

### 5.4 输入法（IME）发送冲突修复

- 问题：中文输入法候选上屏时 Enter 被误判为发送。
- 方案：增加 `composition` 状态与延迟复位，组合输入期间禁止发送判定。
- 价值：提升中文用户输入体验，减少误操作。

证据来源：`2026-05-04.md`（15:50）。

### 5.5 导出与预览解耦

- 问题：预览缩放影响导出排版；打印留白与分页断裂。
- 方案：改 `window.print + @media print`，将预览缩放变量与打印态拆分；打印态隐藏无关 UI 并调整 A4 边距与宽度策略。
- 价值：保证“编辑观感”与“导出质量”两套目标可独立优化。

证据来源：`2026-04-25.md`（23:11）。

### 5.6 封面截图工程化

- 问题：颜色空间、hover 态截入、A4 比例不稳定。
- 方案：切换 `html2canvas-pro`，截图前禁用 pointer 事件并双 rAF，按 A4 比例补白。
- 价值：提升列表封面生成稳定性与视觉一致性。

证据来源：`2026-04-25.md`（23:50）。

## 11. 论文写作时可直接复用的一段话（示例）

本系统围绕“结构化简历编辑 + AI 辅助写作”构建三栏式在线编辑器，采用 Resume/Section/Info Layer 的分层模型解耦内容结构与展示逻辑。工程实现中，通过类型化 API 封装、查询缓存键规范化、模块排序持久化与打印导出样式解耦，逐步形成从“模板管理—内容编辑—AI 辅助—导出交付”的闭环流程。针对流式对话、输入法兼容、预览与导出一致性等真实工程问题，系统在多轮迭代中建立了可复现的问题-方案-验证链路，为后续扩展简历诊断与质量评估提供了稳定基础。
